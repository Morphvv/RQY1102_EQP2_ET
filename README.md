# RQY1102_EQP2_ET
Examen transversal; Ingeneria de software

El proyecto es para crear un sistema para gestionar los gastos comunes en el edificio El Mirador. Este edificio tiene 20 pisos y 160 departamentos. La mayoría de las personas que viven allí son dueñas de sus departamentos, pero también hay algunas que los alquilan.

La administración del edificio tiene problemas porque hace todo de manera manual. Esto significa que no puede cobrar los gastos comunes de manera eficiente y no puede controlar quién paga y quién no. Esto ha generado muchos problemas, como que la gente no paga a tiempo y no hay buena comunicación entre la administración y los residentes. Esto afecta el dinero que tiene el edificio y la confianza que la gente tiene en la administración.

Para solucionar esto, se va a crear una plataforma en la web que haga que todo sea más fácil y automático. Esta plataforma permitirá que la administración gestione a los usuarios, los departamentos y los gastos comunes de manera más eficiente. También permitirá que los residentes puedan pagar sus gastos comunes de manera fácil y rápida.

La plataforma tendrá diferentes niveles de acceso, dependiendo de quién la esté usando. La administración, el personal y los residentes tendrán acceso a diferentes partes de la plataforma. Esto se hará con una arquitectura que sea fácil de usar y de mantener.

Se espera que este sistema mejore la manera en que se gestionan los gastos comunes en el edificio. Se espera que se reduzca la cantidad de gente que no paga a tiempo y que se mejore la comunicación entre la administración y los residentes. También se espera que el edificio sea más eficiente y que se gestione de manera más moderna y transparente.
